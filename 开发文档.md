# UNV Viewer 软件开发文档

## 1. 项目概述

### 项目名称
UNV Data Viewer (UNV数据查看器)

### 项目类型
BS架构的UNV文件解析与可视化工具

### 核心功能
- 解析Universal File Format (UNV) 文件
- 在浏览器中以曲线图形式显示数据
- 支持X/Y轴线性/对数切换
- SQLite数据库存储和批量导入

## 3. 功能说明

### 3.1 文件浏览模式
- **上传文件**: 点击上传区域或拖拽文件
- **测试数据**: 预置测试文件列表
- **数据集选择**: 下拉菜单 + 左右导航按钮

### 3.2 曲线显示
- **数据类型支持**:
  - 函数数据(类型58): 显示实部/虚部/幅值曲线
  - 节点数据(类型15/2411): 显示X/Y/Z坐标
  - 其他类型: 显示类型信息
- **坐标轴模式**: 线性/对数切换

### 3.3 数据库功能
- **批量入库**: 输入文件夹路径导入所有UNV文件
- **部分入库**: 选择文件夹后勾选要导入的文件
- **数据管理**: 查看、删除已入库文件

### 3.4 多文件对比功能 (v3新增)
- **文件浏览模式**: 支持多文件上传，勾选文件后进行通道对比
- **数据库模式**: 支持勾选已入库文件进行通道对比
- **对比操作**: 
  - 勾选2个或以上文件
  - 点击"开始对比"按钮
  - 使用L/R按钮或下拉菜单切换通道
  - 支持线性/对数坐标切换
- **退出对比**: 点击"退出对比"返回单文件模式
- **批量入库**: 输入文件夹路径导入所有UNV文件
- **部分入库**: 选择文件夹后勾选要导入的文件
- **数据管理**: 查看、删除已入库文件

## 4. API接口

### 4.1 解析接口

#### POST /api/parse
上传文件解析

#### POST /api/parse-path
从路径解析文件

#### GET /api/testfiles
获取测试文件列表

### 4.2 数据库接口

#### GET /api/db/stats
获取数据库统计信息

#### GET /api/db/files
获取已入库文件列表

#### GET /api/db/file/:id
获取文件详细信息

#### GET /api/db/dataset/:id
获取数据集曲线数据

#### POST /api/db/import-folder
批量导入文件夹

#### POST /api/db/select-files
导入选中的文件列表

#### DELETE /api/db/delete-file/:id
删除文件记录

#### POST /api/db/folder-files
获取文件夹中的UNV文件列表

## 5. 数据库结构

### files表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| file_path | TEXT | 文件路径 |
| file_name | TEXT | 文件名 |
| file_size | INTEGER | 文件大小 |
| import_time | TIMESTAMP | 入库时间 |
| total_datasets | INTEGER | 数据集数量 |
| total_points | INTEGER | 数据点总数 |

### datasets表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| file_id | INTEGER | 关联files表 |
| dataset_number | INTEGER | 数据集编号 |
| dataset_name | TEXT | 数据集名称 |
| data_points | INTEGER | 数据点数 |
| num_pts | INTEGER | 原始数据点数 |

### function_data表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| dataset_id | INTEGER | 关联datasets表 |
| x_value | REAL | X坐标 |
| y_real | REAL | Y实部 |
| y_imag | REAL | Y虚部 |

## 9. 已知问题

1. **部分UNV文件解析失败**: pyuff库对某些特殊格式支持不完善（如test6.unv）
2. **浏览器安全限制**: input type="file" webkitdirectory无法获取文件夹完整路径，需要用户手动输入
3. **大数据文件**: 超过50MB的文件解析可能较慢
