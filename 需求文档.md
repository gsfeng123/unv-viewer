# UNV数据查看器 - 需求文档

## 1. 项目背景

### 1.1 需求来源
用户需要开发一个BS架构的程序，用于读取和显示UNV（Universal File Format）数据格式的文件。

### 1.2 原始需求
- 读取UNV格式数据
- 在网页浏览器中打开和显示数据
- UNV格式说明文档位置：C:\code\unv_read\docs
- 测试数据位置：C:\code\unv_read\testdata

---

## 2. 功能需求

### 2.1 核心功能
1. **UNV文件解析**
   - 支持解析UNV文件格式
   - 识别各种数据集类型（节点、单元、函数等）
   - 自动读取文件中的数据点数信息

2. **数据可视化**
   - 以曲线图形式显示数据（不是表格）
   - 支持X轴和Y轴的线性/对数模式切换
   - 函数数据显示：实部、虚部、幅值三条曲线
   - 节点数据显示：X、Y、Z坐标曲线

3. **文件操作**
   - 支持上传UNV文件
   - 支持从测试数据目录选择文件

### 2.2 用户界面
1. **数据集导航**
   - 不使用平铺按钮方式
   - 使用下拉菜单选择数据集
   - 添加左右箭头按钮切换数据集
   - 显示当前位置（如：1 / 26）

2. **坐标轴控制**
   - X轴：线性/对数 切换
   - Y轴：线性/对数 切换

### 2.3 数据库功能
- 批量入库：输入文件夹路径导入所有UNV文件
- 部分入库：选择文件夹后手动勾选部分文件入库
- 数据管理：查看、删除已入库文件
- 统计信息：文件数、数据集数、数据点数、总大小

---

## 3. 技术方案

### 3.1 技术选型
- **后端**：Python Flask + pyuff库（解析UNV文件）
- **前端**：HTML + JavaScript + Chart.js（曲线图表）
- **数据库**：SQLite（存储入库数据）

### 3.2 项目结构
```
C:\code\unv_read\
├── server.py           # Flask后端
├── public/
│   └── index.html     # 前端页面

├── docs/              # UNV格式文档

```

## 6. UNV数据集类型

| 编号 | 名称 | 说明 |
|------|------|------|
| 15 | Nodes | 节点 |
| 55 | Data at Nodes | 节点数据 |
| 58 | Function / Group | 函数数据 |
| 82 | Tracelines | 轨迹线 |
| 151 | Project Information | 项目信息 |
| 164 | Response Function | 响应函数 |
| 2411 | Nodes - Double Precision | 双精度节点 |
| 2412 | Elements | 单元 |

## 7. 已知问题

### 7.1 解析问题
- 部分UNV文件（如test6.unv）解析失败
- 原因：pyuff库对某些特殊格式支持不完善

### 7.2 浏览器限制
- input type="file" webkitdirectory 无法获取文件夹完整路径
- 解决方案：使用prompt让用户手动输入文件夹路径

### 7.3 性能问题
- 超过50MB的大文件解析较慢
