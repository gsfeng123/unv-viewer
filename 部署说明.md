# UNV Viewer 部署说明文档

## 1. 环境要求

### 1.1 硬件要求
- CPU: Intel Core i5 或同等性能以上
- 内存: 4GB 以上
- 硬盘: 至少 1GB 可用空间
- 显示器: 1024x768 分辨率以上

### 1.2 软件要求
- 操作系统: Windows 10/11 或 Windows Server 2019+
- Python: 3.8 或更高版本
- 浏览器: Chrome 90+ / Edge 90+ / Firefox 88+

---

## 2. 安装步骤

### 2.1 安装 Python

1. 访问 Python 官网: https://www.python.org/downloads/
2. 下载 Python 3.8 或更高版本
3. 运行安装程序，**勾选 "Add Python to PATH"**
4. 验证安装:
   ```cmd
   python --version
   ```

### 2.2 安装依赖库

打开命令提示符（CMD），执行:

```cmd
pip install pyuff flask flask-cors numpy
```

**注意**: 如果速度慢，可以使用国内镜像:
```cmd
pip install pyuff flask flask-cors numpy -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 2.3 获取项目文件

将项目文件夹 `C:\code\unv_read\` 复制到目标服务器或本地目录。

---

## 3. 启动服务

### 3.1 方式一: 命令行启动

1. 打开命令提示符（CMD）
2. 进入项目目录:
   ```cmd
   cd C:\code\unv_read
   ```
3. 启动服务:
   ```cmd
   python server.py
   ```
4. 看到以下输出表示启动成功:
   ```
   UNV Viewer 服务已启动: http://localhost:3000
   数据库路径: C:\code\unv_read\unv_data.db
   ```

### 3.2 方式二: 开机自启动

创建启动脚本 `start.bat`:

```batch
@echo off
cd /d C:\code\unv_read
python server.py
pause
```

双击 `start.bat` 即可启动。

### 3.3 方式三: Windows 服务（高级）

可以使用 NSSM 将 Python 程序注册为 Windows 服务，实现开机自启动。

---

## 4. 访问软件

### 4.1 本地访问

打开浏览器，访问:
```
http://localhost:3000
```

### 4.2 局域网访问

1. 确认防火墙允许 Python 通过
2. 在浏览器中访问服务器 IP:
   ```
   http://192.168.x.x:3000
   ```
   (将 x.x 替换为实际 IP 地址)

### 4.3 端口说明

- 默认端口: **3000**
- 如需修改，编辑 `server.py` 最后一行:
  ```python
  app.run(host='0.0.0.0', port=3000, debug=True)
  ```
  将 `port=3000` 改为其他端口号。

---

## 5. 数据说明

### 5.1 数据库位置
- 默认: `C:\code\unv_read\unv_data.db`
- 这是一个 SQLite 数据库文件
- 可以复制备份

### 5.2 测试数据
项目已包含测试数据:
- `testdata/` - 17个测试文件
- `unv/psd/` - 6个PSD测试文件

### 5.3 数据导入
1. 点击"数据库"标签
2. 点击"导入全部"按钮
3. 在弹窗中输入文件夹路径（如 `C:\code\unv_read\testdata`）
4. 等待导入完成

---

## 6. 常见问题

### 6.1 端口被占用

如果提示端口 3000 被占用:
```cmd
# 查找占用进程
netstat -ano | findstr :3000

# 结束进程 (需要管理员权限)
taskkill /PID <进程ID> /F
```

### 6.2 依赖安装失败

确保已安装 Visual C++ Build Tools:
- 下载: https://visualstudio.microsoft.com/visual-cpp-build-tools/

### 6.3 浏览器无法访问

1. 检查服务是否启动
2. 检查防火墙设置
3. 尝试使用 localhost 访问

### 6.4 数据解析失败

部分 UNV 文件格式不标准，解析可能会失败，这是 pyuff 库的已知限制。

---

## 7. 卸载/停止

### 7.1 停止服务

在运行服务的命令行窗口按 `Ctrl + C`

### 7.2 完全卸载

1. 停止服务
2. 删除项目文件夹
3. （可选）卸载 Python
